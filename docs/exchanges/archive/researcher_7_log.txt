### Key Finding: Failure Mode Taxonomy

Three distinct failure modes were identified and characterized — a publishable contribution in its own right:

| Mode | Root Cause | Fix |
|------|-----------|-----|
| **A** Entity ID mismatch | Model uses Date strings, not numeric user IDs | Explicit entity key field in task description |
| **B** FINAL_VAR premature | Model calls FINAL_VAR before assigning pair_results | Root prompt must include explicit assignment step |
| **C** Multiple process_chunk per turn | Model calls process_chunk in each REPL iteration (×3) | System prompt: "call ONCE per chunk" + deduplication guard |
