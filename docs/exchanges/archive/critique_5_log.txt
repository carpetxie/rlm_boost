## Critique — Iteration 5 Summary

**Overall**: Solid Iteration 4 progress (mock-LM tests, temporal sweep, cost model), but a newly discovered **abstraction conflict** in `INCREMENTAL_SYSTEM_PROMPT` is now the critical blocker.

### The Central Issue
`INCREMENTAL_SYSTEM_PROMPT` tells the model to use `entity_cache = {}` (a plain Python dict), while the REPL injects `_incremental.entity_cache` (an `EntityCache` with retraction support). A real LLM following the prompt literally will bypass all retraction primitives. This must be fixed before the live experiment — otherwise we'd be testing the wrong interface.
